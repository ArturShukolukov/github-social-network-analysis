# github-social-network-analysis
Анализ социальной сети GitHub с использованием методов теории сложных сетей: распределение степеней, обнаружение сообществ (Louvain), устойчивость сети и пересвязывание рёбер. Итоговая работа по дисциплине «Структура и функции сложных сетей».

## Данные

Используется публичный датасет **GitHub Social Network**:

* **Источник**: SNAP (Stanford Network Analysis Project)
* **Авторы датасета**: B. Rozemberczki, C. Allen, R. Sarkar
* **Год**: 2019
* **Ссылка**: [https://snap.stanford.edu/data/github-social.html](https://snap.stanford.edu/data/github-social.html)

### Описание сети

* **Вершины**: 37 700 пользователей GitHub
  (каждый пользователь поставил «звёздочку» минимум 10 репозиториям)
* **Рёбра**: 289 003 неориентированных связей
  (взаимная подписка между пользователями)
* **Тип сети**: неориентированная, невзвешенная
* **Атрибуты вершин**:

  * `0` — Web-разработчик
  * `1` — ML-разработчик

Для воспроизведения анализа необходимо скачать файлы:

* `musae_git_edges.csv`

## Используемые методы

В ходе работы применяются следующие методы анализа сложных сетей:

### Базовые структурные метрики

* плотность сети
* средняя степень
* диаметр
* средняя длина пути
* коэффициент кластеризации
* ассортативность по степени

### Распределение степеней

* кумулятивное распределение степеней (CCDF)
* визуальная диагностика тяжёлого хвоста
* оценка параметра степенного распределения (γ) методом максимального правдоподобия (`poweRlaw`)

### Кластеризация и сообщества

* обнаружение сообществ алгоритмом **Louvain**
* оценка модульности
* анализ состава сообществ по типам разработчиков (Web / ML)

### Надёжность сети (Robustness)

* случайное удаление вершин (random failures)
* целенаправленная атака на хабы (targeted attack)
* анализ доли вершин в крупнейшей компоненте связности (GCC):

  * относительно оставшихся вершин
  * относительно исходного размера сети

### Пересвязывание рёбер

* случайное пересвязывание рёбер с сохранением степеней (`keeping_degseq`)
* анализ устойчивости гигантской компоненты к структурным перестройкам

## Основные результаты

* Сеть GitHub обладает **свойствами малого мира**: короткие средние пути при большом диаметре, обусловленном периферийными ветвями.
* Распределение степеней имеет **тяжёлый хвост**; оценка параметра γ ≈ 2.5 указывает на scale-free характер сети.
* Сеть является **дисассортативной**: высокостепенные узлы (хабы) преимущественно связаны с узлами малой степени.
* Алгоритм Louvain выявляет **умеренную модульность** (~0.45) и наличие как крупных, так и малых сообществ.
* В большинстве крупных сообществ преобладают **Web-разработчики**, тогда как ML-разработчики распределены более равномерно.
* Сеть **устойчива к случайным отказам**, но **уязвима к целенаправленному удалению хабов**.
* Пересвязывание рёбер при сохранении степеней практически не разрушает гигантскую компоненту.

## Воспроизводимость

1. Установить необходимые пакеты R:

   * `igraph`
   * `poweRlaw`
2. Скачать файлы датасета со страницы SNAP.
3. Поместить CSV-файлы в корень проекта.
4. Запустить основной скрипт анализа.

---

## Ограничения и замечания

* Алгоритм Louvain является **стохастическим**; для воспроизводимости используется фиксированный `set.seed()`.
* Атака на хабы реализована как **фиксированное удаление вершин по степени исходного графа** (без адаптивного пересчёта).
* Анализ robustness носит **структурный характер** и не моделирует динамику процессов на сети.
